# 📋 ПОДРОБНЫЙ ПЛАН: Раздел "Промпты" в GUI

## 🎯 ЦЕЛЬ
Создать полнофункциональный раздел "Промпты" который:
- Точно повторяет функциональность консольного меню (опция 8)
- Имеет красивый и удобный интерфейс
- Показывает полные результаты операций как в оригинале

---

## 🔍 АНАЛИЗ ТЕКУЩИХ ПРОБЛЕМ

### ❌ Проблемы интерфейса (по фото):
1. **Кривая верстка** - элементы наезжают друг на друга
2. **Плохие отступы** - текст прилипает к краям
3. **Неправильные пропорции** - левая панель слишком узкая
4. **Некрасивые кнопки** - кнопка "Запустить" выглядит убого
5. **Плохая типографика** - мелкий текст, плохие шрифты

### ❌ Проблемы функциональности:
1. **Нет полного вывода** - после операций не показывается результат как в консоли
2. **Нет логов** - пользователь не видит что происходит
3. **Нет статистики** - не показывается сколько файлов обработано
4. **Нет ошибок** - если что-то пошло не так, пользователь не знает что

## 🔧 ТЕХНИЧЕСКАЯ АРХИТЕКТУРА

### 📁 Структура файлов:
```
gui/views/prompts/
├── prompts_widget.py          # Главный виджет
├── prompts_left_panel.py      # Левая панель с действиями
├── prompts_right_panel.py     # Правая панель с просмотром
├── prompts_result_panel.py    # Панель результатов операций
└── prompts_dialogs.py         # Диалоги для ввода данных
```

### 🔄 Поток выполнения операций:
```
1. Пользователь выбирает действие → показывается описание
2. Нажимает "Запустить" → появляется диалог ввода (если нужен)
3. Подтверждает → запускается операция в фоне
4. Показывается прогресс → обновляется статус
5. Операция завершается → показывается полный результат
6. Пользователь может скопировать лог или сохранить результат
```
